# MediaTrans 专业版：音视频处理工具需求文档

## 1. 项目愿景
开发一款运行在 Windows 7 及更高版本系统上的现代化音视频处理工具。它不仅具备强大的转换与编辑能力，还拥有丝滑的交互体验（如拖拽裁剪、波形图实时预览），并通过离线激活机制实现商业化闭环。

---

## 2. 核心功能需求

### 2.1 音视频转换与提取
*   **全格式支持**：基于 FFmpeg 内核，支持 MP4, AVI, MKV, MP3, WAV, FLAC 等主流格式互转。
*   **智能提取**：支持“仅保留视频（静音）”或“仅提取音频（视频转 MP3/AAC）”。
*   **参数定制**：支持分辨率、比特率、帧率的自定义调节。

### 2.2 交互式编辑器（核心亮点）
*   **手动拖拽裁剪**：用户在波形图/时间轴上通过左右控制柄（Handles）直接拖拽确定剪辑区间。
*   **波形图渲染**：
    *   **异步加载**：导入文件后立即渲染低采样率预览，后台逐步完善高精度波形。
    *   **缩放平移**：支持鼠标滚轮以光标为中心缩放波形，支持按住右键平移时间轴。
*   **多段拼接**：
    *   支持将多个素材拖入时间轴。
    *   **磁吸功能 (Snapping)**：片段移动时自动对齐，确保拼接处无缝衔接。
*   **音量控制**：支持 -20dB 至 +20dB 的实时增益调整，并在波形图上实时反馈振幅变化。

---

## 3. 离线收费与授权机制

### 3.1 授权逻辑
*   **非对称加密体系**：采用 **RSA-2048** 算法。
    *   **私钥 (Private Key)**：由开发者永久物理保存，严禁丢失，用于签发激活码。
    *   **公钥 (Public Key)**：硬编码在客户端中，用于校验激活码。
*   **激活流程**：
    1.  **机器码生成**：软件通过采集 `CPU ID + 硬盘序列号 + 主板序列号` 经 SHA256 哈希生成唯一机器码。
    2.  **激活码生成**：开发者使用私钥对“机器码 + 授权版本 + 过期时间”进行加密签名。
    3.  **本地校验**：客户端解密后，若机器码匹配且当前日期在有效期内，则解锁功能。

### 3.2 付费墙设计
*   **免费版限制**：导出文件长度限制（如 1 分钟）、添加品牌水印、禁用无损格式导出。
*   **专业版解锁**：去除水印、支持 4K/无损导出、开启多线程加速。

---

## 4. UI/UX 视觉与交互设计

*   **视觉风格**：深色模式 (Dark Theme) 配合高对比度操作区。
*   **Win7 兼容性优化**：
    *   使用自定义窗口边框（摒弃原生 Aero 边框），提升现代感。
    *   **硬件加速**：启用 WPF/GPU 渲染，确保波形图在旧系统上不卡顿。
*   **操作反馈**：
    *   **实时预览**：拖动时间轴时，视频窗口需实时显示对应帧。
    *   **进度感知**：转换过程中显示精确进度条、剩余时间及实时日志输出。

---

## 5. 技术架构规范

| 维度 | 规范要求 |
| :--- | :--- |
| **开发语言** | C# (WPF) 或 C++ (Qt) |
| **目标框架** | .NET Framework 4.5.2 (确保 Win7 SP1 无缝运行) |
| **处理引擎** | FFmpeg 静态编译版（需集成在安装包内） |
| **绘图引擎** | SkiaSharp 或 GDI+ (双缓冲优化) |
| **安全方案** | 代码混淆 (ConfuserEx) + RSA 离线授权 |

---

## 6. 开发阶段划分 (Milestones)

1.  **第一阶段 (MVP)**：搭建 UI 框架，集成 FFmpeg 基础转换功能。
2.  **第二阶段 (编辑器)**：开发波形图渲染算法，实现手动拖拽裁剪与预览。
3.  **第三阶段 (授权系统)**：实现机器码提取、RSA 校验逻辑及付费墙 UI。
4.  **第四阶段 (优化)**：进行 Win7/10/11 多平台兼容性测试与性能调优。

---

## 7. 风险评估
*   **兼容性风险**：Win7 缺少某些多媒体组件，必须完全依赖 FFmpeg 自带解码。
